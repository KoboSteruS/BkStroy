# Управление фотографиями слайдера "Как мы работаем"

## Описание функциональности

Добавлена возможность управлять изображениями для вертикального слайдера в секции "Как мы работаем" через админ-панель.

## Что реализовано:

### 1. Backend (app.py)
- Обновлён endpoint `/api/admin/upload` для поддержки двух типов загрузки:
  - `type=catalog` - для каталога объектов (по умолчанию)
  - `type=slider` - для изображений слайдера процесса
- Изображения слайдера сохраняются в `static/images/` с префиксом `process_`
- Добавлено поле `slider_images` в дефолтную структуру `site_content.json`

### 2. Frontend - Админ-панель

#### HTML (templates/admin.html)
- Добавлена новая секция "Фотографии слайдера 'Как мы работаем'"
- Создано модальное окно для управления изображениями
- Показывается количество этапов работы (должно соответствовать количеству фото)

#### JavaScript (static/js/admin.js)
Добавлены функции:
- `openSliderModal()` - открытие модального окна с текущими изображениями
- `renderSliderImages()` - отрисовка списка изображений с превью
- `onSliderImagesSelected()` - загрузка новых изображений
- `saveSliderImages()` - сохранение изменений
- Кнопка удаления отдельных изображений

#### CSS (static/css/admin.css)
- `.admin-slider-images-list` - сетка для превью изображений
- `.admin-slider-image-item` - карточка изображения
- `.admin-slider-image-controls` - контролы (номер этапа + кнопка удаления)

### 3. Frontend - Главная страница

#### HTML (templates/index.html)
- Обновлён шаблон слайдера для использования массива `slider_images`
- Каждому слайду соответствует своё изображение
- Если изображений меньше, чем этапов - используется первое изображение

#### Jinja2 логика:
```jinja2
{% set slider_images = content.process.get('slider_images', ['images/process.jpg']) %}
{% for step in content.process.steps|default([]) %}
    {% set img_index = loop.index0 if loop.index0 < slider_images|length else 0 %}
    <img src="{{ url_for('static', filename=slider_images[img_index]) }}" ...>
{% endfor %}
```

### 4. Данные (data/site_content.json)
Добавлено поле:
```json
"process": {
  ...
  "slider_images": [
    "images/process.jpg"
  ],
  ...
}
```

## Как использовать:

1. Войти в админ-панель: `/{admin_token}/admin`
2. Нажать кнопку "Управление фотографиями" в секции слайдера
3. Загрузить нужные изображения (количество = количество этапов)
4. Удалить ненужные изображения кнопкой "×"
5. Нажать "Сохранить изменения"
6. Обновить главную страницу для просмотра

## Технические детали:

### Формат загружаемых изображений:
- Поддерживаются: `.jpg`, `.jpeg`, `.png`, `.webp`
- Имена файлов: `process_{uuid}.{ext}`
- Сохраняются в: `static/images/`

### Структура данных:
```javascript
{
  "process": {
    "slider_images": [
      "images/process_abc123.jpg",
      "images/process_def456.jpg",
      ...
    ]
  }
}
```

### Особенности:
- При сохранении текстов сайта `slider_images` сохраняется автоматически
- Если изображений нет, используется дефолтное `images/process.jpg`
- На мобильных изображение используется как фон для этапов
- На десктопе - вертикальный слайдер с fade-эффектом

## Безопасность:
- Все операции требуют авторизации в админ-панели
- Проверка типов файлов на сервере
- UUID в именах файлов предотвращает конфликты
