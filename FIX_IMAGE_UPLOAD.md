# Исправление проблемы с загрузкой фотографий

## Проблема
Клиент не мог загружать фотографии в админ-панели. Ошибка: "The string did not match the expected pattern"

## Причины
1. **Отсутствовала папка `static/uploads/catalog/`** - сервер не мог сохранить файлы
2. **Относительные URL вместо абсолютных** - браузер (особенно мобильный) считал их невалидными
3. **Недостаточная обработка ошибок** - непонятно было, что именно пошло не так

## Что исправлено:

### 1. Backend (app.py)
- ✅ Добавлена обработка ошибок с `try-catch`
- ✅ Сервер теперь возвращает **полные абсолютные URL** с `_external=True`
- ✅ Теперь возвращаются URL вида: `http://domain.com/static/uploads/catalog/xxx.jpg`

### 2. Frontend (static/js/admin.js)
- ✅ Упрощена обработка URL (не нужно преобразование)
- ✅ Улучшены сообщения об ошибках
- ✅ Для слайдера корректно извлекается относительный путь

### 3. Структура файлов
- ✅ Создана папка `static/uploads/catalog/`
- ✅ Добавлен `.gitkeep` для сохранения структуры в Git

## Что теперь происходит:

1. Клиент нажимает "Загрузить фото"
2. Файлы отправляются на `/api/admin/upload`
3. Сервер сохраняет в `static/uploads/catalog/{uuid}.jpg`
4. Возвращается полный URL: `http://your-domain.com/static/uploads/catalog/123.jpg`
5. URL вставляется в textarea (для каталога) или массив (для слайдера)
6. При сохранении URL сохраняется как есть в JSON

## Как проверить:

1. Перезапустить сервер Flask
2. Зайти в админ-панель
3. Попробовать загрузить фото в каталог
4. Должно работать без ошибок

## Если всё равно не работает:

1. Проверьте права на папку `static/uploads/catalog/` (должна быть доступна для записи)
2. Проверьте логи сервера - теперь там будет детальная информация об ошибках
3. Убедитесь, что размер файла не превышает лимиты Flask (по умолчанию 16MB)
